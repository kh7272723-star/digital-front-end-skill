# SECDED ECC patterns (Hsiao code)

## Source authority

This file distills patterns from **lowRISC OpenTitan `prim_secded`** (`github.com/lowrisc/opentitan`, Apache 2.0 license), which implements the **Hsiao code** — a class of SECDED (Single Error Correction, Double Error Detection) codes based on:

> M. Y. Hsiao, "A Class of Optimal Minimum Odd-weight-column SEC-DED Codes," IBM Journal of Research and Development, vol. 14, no. 4, pp. 395–401, July 1970.

OpenTitan's Python generator (`util/design/secded_gen.py`) produces synthesizable SystemVerilog encoder/decoder modules. The patterns here are adapted to plain Verilog with the skill's naming conventions.

## Key property of Hsiao codes

Every column of the parity check matrix **H** has an odd number of 1s. This gives:

| Condition | Syndrome weight | Action |
|-----------|----------------|--------|
| s = 0 | — | No error |
| s ≠ 0, popcount(s) is odd | Single error | Correct the bit at position matching syndrome |
| s ≠ 0, popcount(s) is even | Double error | Flag error, do NOT correct (correction would introduce third error) |

No separate overall parity bit is needed — the odd-weight-column property provides both SEC and DED in one structure.

## Standard widths

| Code (n,k) | Data bits | Check bits | Codeword | Overhead | Typical use |
|------------|-----------|------------|----------|----------|-------------|
| (22,16) | 16 | 6 | 22 | 37.5% | Small register files |
| (39,32) | 32 | 7 | 39 | 21.9% | 32-bit data paths |
| (72,64) | 64 | 8 | 72 | 12.5% | DDR4/DDR5 ECC DIMM |

## 1. Hsiao SECDED encoder — interface and architecture

The encoder computes check bits from data bits using a generator matrix derived from the Hsiao H-matrix. Each check bit is the XOR of a specific subset of data bits.

```verilog
module hsiao_secded_39_32_enc (
  input  wire [31:0] data_i,
  output wire [38:0] codeword_o   // {data_i[31:0], check[6:0]}
);

  // check bits: each is XOR of selected data bits per Hsiao H-matrix
  // The exact XOR subsets are generated from the H-matrix.
  // Below: representative structure for the 32→39 code.
  // In practice: generate from OpenTitan's secded_gen.py or equivalent tool.

  wire [6:0] check;

  // Each check bit C[j] = XOR of data bits where H[j][i] == 1
  // H-matrix is 7 rows × 39 columns (7 check columns + 32 data columns)
  // For a production design: replace these with the actual XOR equations.

  assign check[0] = ^data_i & 32'hXXXX_XXXX;  // placeholder — use actual H row
  assign check[1] = ^data_i & 32'hXXXX_XXXX;
  assign check[2] = ^data_i & 32'hXXXX_XXXX;
  assign check[3] = ^data_i & 32'hXXXX_XXXX;
  assign check[4] = ^data_i & 32'hXXXX_XXXX;
  assign check[5] = ^data_i & 32'hXXXX_XXXX;
  assign check[6] = ^data_i & 32'hXXXX_XXXX;

  assign codeword_o = {data_i, check};

endmodule
```

**Important**: The XOR equations above are placeholders. The actual H-matrix values for each code size must be generated by a tool (OpenTitan's `secded_gen.py`, or a custom script). The skill does not hardcode the H-matrix — it would be error-prone and untestable.

The canonical way to obtain the H-matrix:
```bash

cd opentitan
python3 util/design/secded_gen.py --outdir ./ecc_rtl
# Generates: prim_secded_39_32_enc.sv, prim_secded_39_32_dec.sv,
#             prim_secded_72_64_enc.sv, prim_secded_72_64_dec.sv, etc.
```

## 2. Hsiao SECDED decoder — syndrome + correction

```verilog
module hsiao_secded_39_32_dec (
  input  wire [38:0] codeword_i,
  output wire [31:0] data_o,
  output wire [6:0]  syndrome_o,
  output wire [1:0]  err_o         // [0]=single_err, [1]=double_err
);

  // syndrome = H × codeword_i  (XOR operations)
  wire [6:0] syndrome;
  // PLACEHOLDER: each syndrome bit is XOR of selected codeword bits per H-matrix
  assign syndrome[0] = ^codeword_i & 39'hXX_XXXX_XXXX;  // replace with actual H row
  // ... remaining 6 syndrome bits

  // popcount of syndrome (determines odd/even weight)
  wire syndrome_odd;
  assign syndrome_odd = ^syndrome;  // XOR reduction: 1 = odd parity

  // syndrome == 0 → no error
  wire no_err;
  assign no_err = (syndrome == 7'd0);

  // single error: syndrome non-zero AND odd weight
  wire single_err;
  assign single_err = !no_err && syndrome_odd;

  // double error: syndrome non-zero AND even weight
  wire double_err;
  assign double_err = !no_err && !syndrome_odd;

  // error position decode: syndrome matches H column K → error at bit K
  // PLACEHOLDER: one comparator per codeword bit position
  wire [38:0] err_pos;
  // assign err_pos[0] = (syndrome == 7'bxxx_xxx);  // H column 0
  // ... 39 comparators

  // correction: flip the bit at err_pos
  wire [38:0] corrected;
  assign corrected = codeword_i ^ (err_pos & {39{single_err}});

  assign data_o    = corrected[31:0];
  assign syndrome_o = syndrome;
  assign err_o[0]  = single_err;
  assign err_o[1]  = double_err;

endmodule
```

Pattern rule:
- The decoder is **purely combinational** — typical critical path is the XOR tree for syndrome + comparator tree for error location.
- For high-frequency designs, pipeline the decoder: syndrome → register → error-locate → register → correct.
- `err_o[0]` (single) and `err_o[1]` (double) are mutually exclusive.
- When `double_err=1`, `data_o` is the uncorrected data — do NOT use it.
- The syndrome and error-position comparators must be generated from the H-matrix, not hand-coded.

## 3. Pipelined ECC wrapper (for timing closure)

```verilog
module hsiao_secded_39_32_pipe #(
  parameter PIPE_STAGES = 1   // 0=combinational, 1=1-stage, 2=2-stage
) (
  input  wire        clk,
  input  wire        rst,

  input  wire        valid_i,
  input  wire [31:0] data_i,
  output wire [38:0] codeword_o,

  input  wire [38:0] codeword_i,
  output wire        valid_o,
  output wire [31:0] data_o,
  output wire [1:0]  err_o
);

  // Encoder: combinational, optionally registered
  wire [38:0] enc_raw;
  hsiao_secded_39_32_enc u_enc (.data_i(data_i), .codeword_o(enc_raw));

  // Decoder: always combinational first
  wire [31:0] dec_raw;
  wire [1:0]  err_raw;
  hsiao_secded_39_32_dec u_dec (.codeword_i(codeword_i), .data_o(dec_raw), .syndrome_o(), .err_o(err_raw));

  // Optional pipeline registers
  generate
    if (PIPE_STAGES == 0) begin : g_comb
      assign codeword_o = enc_raw;
      assign valid_o    = valid_i;
      assign data_o     = dec_raw;
      assign err_o      = err_raw;
    end else begin : g_pipe
      reg [38:0] enc_r;
      reg        valid_r;
      reg [31:0] dec_r;
      reg [1:0]  err_r;

      always @(posedge clk) begin
        if (rst) begin
          valid_r <= 1'b0;
        end else begin
          enc_r   <= enc_raw;
          valid_r <= valid_i;
          dec_r   <= dec_raw;
          err_r   <= err_raw;
        end
      end

      assign codeword_o = enc_r;
      assign valid_o    = valid_r;
      assign data_o     = dec_r;
      assign err_o      = err_r;
    end
  endgenerate

endmodule
```

## Synthesis notes

- Hsiao encoder: ~N_data XOR gates with fan-in ~(#check_bits)/2 on average. For (72,64): ~64 XORs, each ~4-input on average.
- Hsiao decoder: syndrome calc (~N_codeword XORs, each K/2 fan-in) + K-to-N decoder comparators (~N_codeword comparators of K-bit each).
- Hsiao is faster than classic Hamming because the H-matrix balances the number of 1s per row → balanced XOR tree depth.
- Maximum combinational depth for (72,64): ~4-5 XOR levels for syndrome, ~4-5 levels for comparators. At 500 MHz, pipeline is typically needed.

## Verification notes

Directed tests:
1. Zero error: valid codeword → syndrome=0, err_o=0, data_o = original.
2. Single-bit error (each bit position): syndrome non-zero odd → err_o[0]=1, data corrected.
3. Double-bit error (all pairs): syndrome non-zero even → err_o[1]=1, data NOT corrected.
4. Adjacent-bit errors: check no false correction.
5. All-zeros and all-ones codewords: verify syndrome behavior.
6. Random error injection (10k patterns): check SEC and DED properties.

Assertions:
```systemverilog
assert property (@(posedge clk) disable iff (rst)
  err_o[0] |-> !err_o[1]);  // single and double are mutually exclusive
assert property (@(posedge clk) disable iff (rst)
  err_o[0] |-> $onehot(err_pos));  // exactly one bit corrected
```

## What to capture from ECC examples
- Hsiao code: odd-weight-column property gives SEC + DED without separate parity bit
- Encoder = XOR tree (combinational), Decoder = syndrome + locate + correct (combinational)
- The H-matrix MUST be tool-generated — do not hand-code the XOR equations
- Pipeline registers break the combinational path for high-frequency designs
- Single error → correct and continue. Double error → flag, do not use data
- Standard widths: (39,32) for 32-bit, (72,64) for 64-bit — matches industry ECC DRAM
- OpenTitan's generator is the verified open-source reference for H-matrix values
